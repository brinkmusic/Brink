# Code review — T00–T08 shipped surface (2026-07-02)

Full-repo review of everything shipped by tickets T00–T08, run as seven parallel review
streams (backend foundation, security slice, routers + tests, migrations + packaging,
frontend auth slice, infra/CI/docs, POC dead-shell audit). **Findings only — no fixes were
applied.** Each finding has a stable ID; the remediation tickets (`docs/plans/tickets/backlog/070`–`078`)
cite these IDs, so this file is the traceability root.

**Scope reviewed:** all of `backend/`; the frontend auth slice (`LoginPage`, `CallbackPage`,
`AuthContext`, `lib/supabase.ts`, `App.tsx`, `main.tsx`, `NavBar`); infra/CI/config
(`render.yaml`, `vercel.json`, `vite.config.ts`, both `package.json`, `ci.yml`, `.env.example`);
plus a shallow "dead? ticketed?" pass over the POC shell (rest of `apps/web/src`).

**Verified clean (no action needed):** SQLModel schema ↔ Alembic baseline ↔ old Prisma schema
parity is exact (14 tables, indexes, FK rules, enums); the AES-256-GCM crypto port is faithful
(format-compatible with TS-era blobs); T08 left no TS/Prisma stragglers in code, CI, or config;
nothing in the POC shell is both dead and untracked by a backlog ticket; `lru_cache` singletons,
`normalize_url`, and the pre-commit secret guard regexes are all correct.

## Patterns to standardize before T10–T14

1. **Envelope-complete error handling first** (T070) — the single biggest pre-T10 item.
2. **Strict Pydantic request schemas** as ADR-0007 layer 1; capture-spotify's all-Optional
   pattern is a documented parity exception, not the template (T074 documents this).
3. **One shared frontend API helper** (`apiFetch`) that attaches the Supabase JWT, checks
   `res.ok`, throws typed errors (T075) — three review streams independently converged on this.
4. **`backend/tests/conftest.py`** + real-DB (in-memory SQLite) tests for upsert-style logic
   (T073) — the MagicMock style already missed an entire branch.
5. **Router prefix convention** — `APIRouter(prefix="/api/<resource>", tags=[...])` per
   resource file from T010 onward (noted in T070).
6. **Response DTO / casing decision is due at T010** — no serialization schema exists; raw
   SQLModel returns would emit snake_case JSON and leak columns (`User.email`;
   `SpotifyToken` must never be serializable). The T06-deferred camelCase-alias decision
   has hit its deadline — resolve in T010 (likely as an ADR).
7. Smaller: shared `utcnow_naive()` helper; document exception contracts on security helpers;
   test doubles must mirror the real dependency's failure mode (raise, not return-None);
   pin runtimes in-repo and use locked installs everywhere (T077); rate limiting (ADR-0007 §5)
   is already scoped into T010's helper — keep it there.
